Style Guide

  • Indentation: Use 2 spaces for indentation.
  • Variables: Use `const` for variables that do not change and `let` for variables that do change.
  • Line Length: Limit lines to 80 characters maximum.
  • Quoting Strings: Use single quotes for strings unless escaping is necessary.
  • Function Declarations: Prefer function expressions over function declarations, and use arrow functions for anonymous functions.
  • Spacing: Use spaces around operators and after commas, and avoid trailing whitespace.
  • Comments: Write descriptive comments for complex code to explain the intent and logic.
  • Object and Array Creation: Use object literals `{}` and array literals `[]` instead of constructors (`new Object()` or `new Array()`).
  • ES6 Features: Utilize ES6 features like template literals, destructuring, and spread/rest operators when appropriate.
  • Imports and Exports: Use named exports and imports for better readability and maintainability.
  • Null and Undefined: Avoid explicit comparisons to `null` and `undefined` by using truthy/falsy checks (`if (variable)` or `if (!variable)`).
  • Implicit Type Coercion: Avoid implicit type coercion and use explicit comparisons (`===` or `!==`).
  • Promises and Async/Await: Prefer `async/await` over plain promises for better readability.
  • Error Handling: Always handle errors properly, and avoid swallowing exceptions.
        
const initialState = {
  users: [],
  loading: false,
  error: null,
};

function leaderboardReducer(state = initialState, action) {
  switch (action.type) {
    case 'FETCH_USERS_REQUEST':
      return {
        ...state,
        loading: true,
        error: null,
      };
    case 'FETCH_USERS_SUCCESS':
      return {
        ...state,
        users: action.payload,
        loading: false,
      };
    case 'FETCH_USERS_FAILURE':
      return {
        ...state,
        loading: false,
        error: action.payload,
      };
    default:
      return state;
  }
}
